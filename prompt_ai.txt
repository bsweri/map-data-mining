## IDENTITAS PROYEK

Nama Aplikasi   : Smart Marketing Tools (Map Data Mining)
Deskripsi       : Aplikasi web SPA (Single Page Application) untuk mengekstraksi
                  data bisnis (nama, alamat, nomor telepon, rating, dan jarak)
                  dari Google Maps berdasarkan keyword, lokasi/area, dan radius
                  pencarian yang ditentukan pengguna. Hasil pencarian ditampilkan
                  dalam tabel interaktif dan dapat diekspor ke file Excel (.xlsx).
Pemilik         : Eriandi Susanto (@bsweri)
URL Live        : https://bsweri.github.io/map-data-mining/
URL Portofolio  : https://bsweri.github.io/
Repository      : https://github.com/bsweri/map-data-mining

### Edge Functions yang sudah di-deploy:
1. search-maps          : Menerima { keyword, location, radius } dari frontend.
                          Melakukan geocoding (jika input berupa nama area) atau

## ALUR KERJA APLIKASI (USER FLOW)

1. Pengguna membuka https://bsweri.github.io/map-data-mining/
2. Pengguna memasukkan keyword bisnis (misal: "Apotek"), lokasi/area (misal:
   "Jakarta Selatan" atau menggunakan tombol GPS untuk deteksi otomatis), dan
   mengatur radius pencarian (1-50 KM) menggunakan slider.
3. Pengguna menekan tombol "Cari Lokasi".
4. Frontend mengirim POST request ke Supabase Edge Function `search-maps`.
5. Edge Function melakukan geocoding lokasi (jika perlu), memanggil Google Maps
   Nearby Search API, lalu mengambil detail telepon dari Place Details API.
6. Data dikembalikan ke frontend dan ditampilkan dalam tabel interaktif.
7. Pengguna dapat mengekspor seluruh data ke file Excel (.xlsx).
8. Pengguna dapat membuka lokasi langsung di Google Maps via kolom "Aksi".
9. Pengguna dapat memberikan donasi via tombol "Dukung Aplikasi Ini" yang
   memunculkan pop-up pembayaran Midtrans Snap.

## CATATAN PENTING UNTUK AI

- Jangan pernah mengubah base path di vite.config.ts (/map-data-mining/)
  kecuali diminta secara eksplisit oleh pemilik.
- Jangan pernah membuat workflow file baru di .github/workflows/ selain
  deploy.yml yang sudah ada, karena dapat menyebabkan konflik deployment.
- Semua perintah git di terminal Windows harus menggunakan cmd.exe /c "..."
  atau PowerShell semicolon (;) sebagai separator, BUKAN && (tidak didukung
  di PowerShell versi lama).
- Untuk menjalankan npm di terminal, gunakan cmd.exe /c "npm ..." jika
  PowerShell menolak eksekusi script.
- Supabase Edge Functions di-deploy menggunakan:
  npx supabase functions deploy <nama-fungsi> --no-verify-jwt
- Frontend tidak menggunakan development lokal (localhost). Semua perubahan
  langsung di-push ke GitHub dan otomatis ter-deploy via GitHub Actions.
- Pastikan setiap perubahan kode lolos TypeScript build (tsc -b) sebelum push,
  karena GitHub Actions akan gagal jika ada error TypeScript.
